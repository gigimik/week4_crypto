- Signature Algorithm: ecdsa-with-SHA256
- Public Key Type / Size: EC P-256 (256 bit)

3) TLS Handshake Explanation (TLS 1.3)
- ClientHello → browser proposes TLS 1.3, cipher suites (AES-GCM, ChaCha20-Poly1305), SNI = github.com.  
- ServerHello → server selects TLS 1.3 and TLS_AES_128_GCM_SHA256 cipher suite, sends its key share.  
- EncryptedExtensions, Certificate, CertificateVerify → server authenticates itself via Sectigo CA.  
- Finished → both parties verify handshake integrity, then application data is encrypted.

4) Protection Against MITM Attacks
- Certificate Validation → browser verifies CA chain to USERTrust Root CA.  
- Hostname Check → SNI must match CN/SAN = github.com.  
- Ephemeral ECDHE → provides forward secrecy (ECDHE key exchange seen as x25519 or prime256v1).  
- AEAD Cipher → AES-GCM protects confidentiality and integrity.  
- TLS 1.3 Handshake Encryption → prevents certificate interception or downgrade attacks.

5) TLS Use in HTTPS
- Authentication → server identity proven via trusted CA chain.  
- Confidentiality → HTTP data encrypted after handshake.  
- Integrity → AEAD detects tampering.  
- Forward Secrecy → compromising keys later cannot reveal past sessions.
